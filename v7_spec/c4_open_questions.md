# V7 实施前决策清单 — 开放问题

## SKU 数据准备范围

### 1. **dimensions.source 最低标准是什么？**
- **问题**: 多少精度的尺寸算"事实"？手量、包装标注、供应商规格表、用户自测？
- **当前状态**: 规格要求有 source 属性，但没定义可接受的来源质量
- **决策需求**: 明确 listing_xlsx, manual_input, sku_field, comm_image_measurement 四种 source 的具体标准
- **影响**: size-spec slot 生成成功率

### 2. **use_cases 边界如何界定？**
- **问题**: listing 提到"厨房刀"，是否可以推断"切菜、切肉、切水果"三个 use_case？
- **当前状态**: 要求 source 可追溯，但推断深度未定义
- **决策需求**: 是只接受明确列举的用途，还是允许合理推断？
- **影响**: use-proof 和 usage-demo slot 内容丰富度

### 3. **material.primary 语言和精度要求？**
- **问题**: "不锈钢"、"304不锈钢"、"нержавеющая сталь"、"stainless steel" 哪个是标准格式？
- **当前状态**: 支持俄文技术术语，但没有标准化格式
- **决策需求**: 建立材质词汇标准化表，支持多语言映射
- **影响**: material-macro slot 一致性和 material_profile 匹配

## 降级策略细节

### 4. **packaging 缺失时是强制 7图还是允许 generic unboxing？**
- **问题**: 没有真实包装参考时，是彻底跳过包装相关内容，还是展示产品本体？
- **当前状态**: C0 规定 BLOCK unboxing-scene，但可能影响套图完整性
- **决策需求**: 7图套图在商业上是否足够，还是需要8图保持竞争力？
- **影响**: 套图商业价值和成本控制

### 5. **多材质产品如何处理？**
- **问题**: 不锈钢刀 + 塑料手柄的产品，material.primary 写什么？如何加载 material_profile？
- **当前状态**: 规格支持单一 primary material，但现实产品常为复合材质
- **决策需求**: 是否需要 material.secondary 字段？如何处理约束冲突？
- **影响**: material-macro slot 准确性

### 6. **use_cases 为空时，usage-demo 能否用 category 默认场景？**
- **问题**: 刀具类目有通用使用方法，但 sku_truth 未列举具体 use_case
- **当前状态**: C0 要求 BLOCK usage-demo，但可能过于严格
- **决策需求**: 是否允许 archetype 级别的默认使用场景？
- **影响**: usage-demo slot 生成成功率

## Critic 评分标准

### 7. **metal_realism 判断标准的主观性问题？**
- **问题**: "luxury enhancement" vs "appropriate quality presentation" 边界模糊
- **当前状态**: 依赖 AI 视觉判断，可能不稳定
- **决策需求**: 是否需要基于 price range 的客观标准？低价产品禁止高光，高价产品允许适当提升？
- **影响**: metal_realism hard gate 误判率

### 8. **dimension_provenance OCR 容错率多少？**
- **问题**: "25mm" vs "25 мм" vs "2.5cm" vs "25毫米" 都算匹配吗？
- **当前状态**: fuzzy_match 容忍5%误差，但单位转换和语言混用未考虑
- **决策需求**: 建立数值等价表，支持单位转换和多语言
- **影响**: dimension_provenance 误杀率

### 9. **use_case_provenance 俄语模糊匹配阈值？**
- **问题**: listing 写"для обрезки"，图上写"обрезание"，算匹配吗？
- **当前状态**: threshold=0.7，但俄语词形变化复杂
- **决策需求**: 是否需要专门的俄语语义匹配模型？
- **影响**: use_case_provenance 准确率

## 成本和性能优化

### 10. **fallback 重试次数上限在实际场景下是否合理？**
- **问题**: hard gate 失败重试2次，可能在困难 SKU 上仍不够
- **当前状态**: 控制成本，但可能影响成功率
- **决策需求**: 基于 SKU 复杂度动态调整重试预算？简单刀具2次，复杂电子产品5次？
- **影响**: 成功率vs成本平衡

### 11. **sku_truth 准备时间目标5min/SKU是否现实？**
- **问题**: 首次建立 sku_truth 可能需要更多时间研究产品
- **当前状态**: 基于理想情况估算
- **决策需求**: 是否需要 sku_truth 模板库？常见类目预填充？
- **影响**: 运营效率和推广阻力

### 12. **critic 新维度计算成本影响？**
- **问题**: 4个新维度可能包含OCR、语义分析，增加单次critic成本
- **当前状态**: 未测试实际计算成本
- **决策需求**: 是否需要维度优先级？先跑cheap维度，失败再跑expensive维度？
- **影响**: 总体成本控制

## 质量保证和监控

- **决策需求**: 是否需要集成 Ozon 合规 API 或规则检测？

- **决策需求**: 需要多少样本量？如何控制变量？测试周期多长？
- **影响**: 商业价值验证可信度

## 决策优先级建议

### **高优先级（必须在 M1-M5 前决定）**
- Q1, Q2, Q3: 数据标准化影响核心实现
- Q4, Q6: 降级策略影响架构设计

### **中优先级（M8 前需要决定）**  
- Q7, Q8, Q9: Critic 调优需要标准
- Q13, Q14: 扩展策略影响 M9 实施
- Q15: 质量标准影响验收

### **低优先级（M10-M12 期间优化）**
- Q11, Q12, Q16-Q20: 运营优化和长期监控

**建议决策方式**: 高优先级问题在 M1 开始前召开专门决策会议；中优先级通过 M4-M8 期间的实践验证确定；低优先级基于 v7 稳定运行后的数据和反馈逐步完善。
